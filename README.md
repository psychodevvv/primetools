# ПрофИнструмент — интернет-магазин инструментов

Интернет-магазин на Django: каталог с категориями, подкатегориями и
фильтрами, корзина, оформление заказов, вход покупателей по SMS-коду,
админ-панель с импортом каталогов (Excel и YML-фиды), уведомления о
заказах в Telegram.

**Стек:** Python 3.12+, Django 4.2, SQLite, чистый HTML/CSS/JS.

---

## Запуск с нуля

### 1. Установить Python
Скачать Python 3.12+ с [python.org](https://www.python.org/downloads/).
При установке отметить галочку **«Add Python to PATH»**.

### 2. Создать виртуальное окружение
В папке проекта открыть терминал:

```bash
python -m venv venv
```

Активировать окружение:
- Windows: `venv\Scripts\activate`
- macOS/Linux: `source venv/bin/activate`

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Настроить файл `.env`
Скопировать пример:

```bash
copy .env.example .env        # Windows
cp .env.example .env          # macOS/Linux
```

Сгенерировать секретный ключ и вставить его в `.env` → `SECRET_KEY=...`:

```bash
python -c "from django.core.management.utils import get_random_secret_key as g; print(g())"
```

### 5. Применить миграции базы данных

```bash
python manage.py migrate
```

### 6. Создать администратора

```bash
python init_project.py
```

Создаётся вход в админку: **admin / admin123** — смените пароль после
первого входа.

### 7. Запустить сервер

```bash
python manage.py runserver
```

- Сайт: <http://127.0.0.1:8000/>
- Админ-панель: <http://127.0.0.1:8000/admin-panel/>

## Настройки `.env`

| Переменная | Назначение |
|---|---|
| `SECRET_KEY` | Секретный ключ Django. Обязательно свой, держать в тайне. |
| `DEBUG` | `True` для разработки, `False` на боевом сервере. |
| `ALLOWED_HOSTS` | Домены сайта через запятую. |
| `TELEGRAM_BOT_TOKEN` | Токен бота (от @BotFather) для уведомлений о заказах. |
| `TELEGRAM_ADMIN_CHAT_ID` | ID чата, куда слать уведомления. |
| `CALL_NUMBER` | Номер для звонков — отображается на сайте. |
| `WHATSAPP_NUMBER` | Номер WhatsApp (только цифры) — кнопка «Написать». |
| `SMS_API_KEY` / `SMS_API_URL` / `SMS_SENDER` | SMS-шлюз для кодов входа. Пусто — демо-режим (код виден на экране). |

## Наполнение каталога

В админке раздел **«Импорт каталога»** принимает:

- **Excel (.xlsx)** — любой прайс-лист, столбцы распознаются автоматически.
- **YML-фид (.yml / .xml)** — формат Yandex Market: название, цена, фото,
  описание, характеристики и дерево категорий загружаются сразу.

Раздел **«Обработка товаров»**:
- распознавание категорий по ключевым словам;
- загрузка фото / описаний / характеристик со страниц поставщика;
- поиск товаров по артикулу на сайте поставщика.

Загрузить фото и описания для всех товаров одной командой:

```bash
python manage.py fetch_images            # только пустые
python manage.py fetch_images --refresh  # перезаписать всё
```

## Вход покупателей

Покупатели входят и регистрируются **по коду из SMS** — пароль не нужен.
Для реальной отправки SMS подключите шлюз (mobizon.kz, smsc.kz и т.п.)
через `SMS_API_*` в `.env`. Без шлюза работает демо-режим: код
показывается на экране (удобно при разработке).

## Админ-панель

- **Заказы** — список, поиск по № заказа / имени / телефону, смена статуса.
- **Покупатели** — список, поиск по ID / имени / телефону, изменение
  данных, удаление.
- **Товары / Категории** — управление каталогом, постраничный вывод.
- **Импорт каталога** — загрузка Excel и YML.
- **Обработка товаров** — категории, фото, описания, поиск на сайте.

## Полезные команды

```bash
python manage.py runserver        # запустить сервер
python manage.py migrate          # применить миграции БД
python manage.py makemigrations   # создать миграции после правок моделей
python manage.py collectstatic    # собрать статику (для боевого сервера)
python manage.py fetch_images     # подтянуть фото/описания товаров
```

## Боевой сервер (коротко)

1. `DEBUG=False` и реальный `ALLOWED_HOSTS` в `.env`.
2. Уникальный `SECRET_KEY`.
3. `python manage.py collectstatic`.
4. Запуск через gunicorn/uwsgi + nginx (не `runserver`).
5. Файлы `.env` и `db.sqlite3` — не выкладывать в публичный доступ
   (уже в `.gitignore`).
